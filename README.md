Este repositorio documenta el diseño y evaluación de un sistema de ayuda a la decisión clínica para el diagnóstico de apnea obstructiva del sueño (AOS) a partir de oximetría nocturna. La práctica se centra en la clasificación binaria de segmentos de saturación de oxígeno en sangre (SpO2) para determinar si corresponden a un evento respiratorio (apnea/hipopnea) o a respiración normal, como alternativa de screening frente a la polisomnografía (PSG), considerada el estándar diagnóstico pero con alta complejidad y coste operativo.

Se utiliza una base de datos etiquetada compuesta por 4208 segmentos de SpO2 de 60 segundos muestreados a 16 Hz (960 muestras por segmento), registrados con pulsioximetría portátil y anotados por un experto en sueño en dos clases: “No Apnea” (0) y “Apnea” (1). El flujo de trabajo incluye carga y análisis descriptivo de los datos, separación de matriz de observaciones y vector de etiquetas, y transformación a formato matricial para facilitar el preprocesado.

Como etapa clave de preprocesado se implementa un resampling para reducir la frecuencia de muestreo de 16 Hz a 1 Hz, agregando cada bloque de 16 muestras mediante su media aritmética, con el objetivo de acelerar el entrenamiento y adaptar la señal a su dinámica de variación lenta. Posteriormente, se realiza estandarización (media 0 y varianza 1) utilizando exclusivamente estadísticas del conjunto de entrenamiento para evitar sesgos, aplicando dichas transformaciones de manera consistente sobre validación y test.

La metodología de evaluación se basa en una partición independiente en tres subconjuntos: entrenamiento (50%), validación (25%) y test (25%), garantizando separación entre optimización de hiperparámetros y evaluación final. Se entrena una red neuronal tipo perceptrón multicapa de una única capa oculta mediante el paquete nnet, optimizando el número de neuronas ocultas (size) y el parámetro de regularización (decay) usando la precisión (Accuracy) como métrica objetivo. Para reducir el efecto de la inicialización aleatoria, la optimización se realiza con múltiples repeticiones y se sintetiza en una matriz de rendimientos por combinación de hiperparámetros, acompañada de su representación gráfica.

Finalmente, se entrena el modelo definitivo con los hiperparámetros óptimos y se evalúa sobre el conjunto de test mediante matriz de confusión y métricas diagnósticas estándar: sensibilidad, especificidad, valor predictivo positivo, valor predictivo negativo y precisión. El repositorio incluye scripts reproducibles para el resampling, estandarización, optimización y evaluación, así como los entregables gráficos y tabulares necesarios para documentar el rendimiento del clasificador.
